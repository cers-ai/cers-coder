### 🔌 API设计规范

#### RESTful原则
- 使用HTTP动词（GET、POST、PUT、DELETE）
- 资源导向的URL设计
- 统一的响应格式

#### 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": "2024-01-01T00:00:00Z"
}
```

#### 错误处理
- 标准HTTP状态码
- 详细的错误信息
- 错误码和错误描述
